About This Application

This application allows a user to upload a file to a remote server and enter the title and description of the uploaded file that is persisted on a back end server. It is designed according to requirements outlined below. Requirement implementations are also noted.

See the Readme file in the dotsub-fileupload-frontend Github repository for more details on the Angular 2 project. The web service back end code is located in the dotsub-fileupload-backend Github repository. There is also a Readme file in that project for details on the Spring Rest Controller back end.

Requirements

A description of a requirement's' implementation is noted in pink besides each requirement.

Create a simple web project, that provides a form to upload a file and gather some metadata for that file: Title, Description and Creation date

  • The UI can be implemented in any SPA framework of your choice. I chose Angular 2
  • The REST back-end will be preferably implemented based on Java/Spring Boot (+ any Spring/Java library/framework of your choice) to store data. Spring Boot web service using RestController and Spring Data JPA for persistence
  • No user authentication and security features are needed. No authentication was done. HTML escaping was done on the back end to help prevent XSS attacks from persisted data.
  • Project can use simple in-memory DB to store data. H2 in-memory database backed by Hibernate was used
  • Files can be stored in web server file system. Uploaded files are stored on web server in upload-dir folder
  • No need to implement routing or sophisticated state management in UI (e.g. FLUX). No state management was used on the front end
  • Project can use any quality assurance mechanisms or build tools of choice. Quality assurance used Jasmine, Karma and Protractor for testing and Tslint on the front end and JUnit and Spring Testing Tools and Checkstyle on the back end. The front-end build employed Webpack while Gradle was used to build the back-end web service.
  • You may also submit with the project any design considerations, restrictions, or possible future enhancements (if applicable) See the Readme files in the front-end and back-end Github repositories for details.

(Optional) Create an additional view where the user can see the list of uploaded items. Each line will show the Title, Description, Creation date and a link to download the file. This view is encapsulated the FileUploadDataTableComponent whose template is displayed on the bottom of the file upload form on the home page.
